# 英语口语学习App - 场景数据模型设计

## 1. 场景主表 (Scene)

### 1.1 表结构
```json
{
  "scene_id": "string",           // 场景唯一标识，如 "daily_001"
  "scene_name": "string",         // 场景名称，如 "餐厅点餐"
  "category": "string",           // 场景分类：daily/workplace/study_abroad
  "difficulty": "string",         // 难度：beginner/advanced
  "duration": "number",           // 学习时长（分钟），固定10
  "description": "string",        // 场景简介
  "tags": ["string"],             // 关键词标签，用于搜索
  "dialogue": "Dialogue",         // 高频对话内容（关联对话对象）
  "vocabulary": ["Vocabulary"],   // 高频单词/短语列表
  "tests": ["Test"],              // 测试题列表
  "created_at": "timestamp",      // 创建时间
  "updated_at": "timestamp"       // 更新时间
}
```

### 1.2 字段说明
- **category枚举值**：
  - `daily`: 日常场景（问候、点餐、购物、问路等）
  - `workplace`: 职场场景（自我介绍、面试问答等）
  - `study_abroad`: 留学/考试场景（雅思口语、留学面签等）

- **difficulty枚举值**：
  - `beginner`: 入门
  - `advanced`: 进阶

## 2. 对话内容表 (Dialogue)

### 2.1 表结构
```json
{
  "dialogue_id": "string",        // 对话唯一标识
  "scene_id": "string",           // 关联场景ID
  "rounds": ["DialogueRound"],    // 对话回合列表（≤5个回合）
  "full_audio_url": "string",     // 完整对话音频URL
  "duration": "number"            // 音频时长（秒）
}
```

### 2.2 对话回合 (DialogueRound)
```json
{
  "round_number": "number",       // 回合序号（1-5）
  "content": ["DialogueContent"], // 该回合的对话内容列表（一问一答或多句）
  "analysis": "QAAnalysis"        // 该回合的问答解析（可选，仅核心问答回合有）
}
```

### 2.3 对话内容 (DialogueContent)
```json
{
  "index": "number",              // 对话内容序号（回合内的顺序）
  "speaker": "string",            // 说话者角色，如 "customer"/"waiter"
  "speaker_name": "string",       // 说话者显示名称，如 "顾客"/"服务员"
  "text": "string",               // 对话文本（英文）
  "translation": "string",        // 中文翻译
  "audio_url": "string",          // 该句音频URL
  "is_key_qa": "boolean"          // 是否为核心问答
}
```

---

## 3. 问答解析 (QAAnalysis)

### 3.1 表结构
```json
{
  "analysis_detail": "string",    // 分析详情说明
  "standard_answer": "Answer",    // 标准回答（对话中的原回答）
  "alternative_answers": ["Answer"], // 其他回答方式（2-3种）
  "usage_notes": "string"         // 使用场景说明
}
```

### 3.2 回答对象 (Answer)
```json
{
  "answer_id": "string",          // 回答唯一标识
  "text": "string",               // 回答文本（英文）
  "translation": "string",        // 中文翻译
  "audio_url": "string",          // 音频URL
  "scenario": "string",           // 适用场景说明
  "formality": "string"           // 正式程度：formal/neutral/casual
}
```

---

## 4. 高频单词/短语表 (Vocabulary)

### 4.1 表结构
```json
{
  "vocab_id": "string",           // 单词/短语唯一标识
  "scene_id": "string",           // 关联场景ID
  "type": "string",               // 类型：word/phrase
  "content": "string",            // 单词或短语内容
  "phonetic": "string",           // 音标，如 "/ˈɔːrdər/" 或 "/ˈredɪ tə ˈɔːrdər/"
  "translation": "string",        // 中文释义（贴合场景）
  "example_sentence": "string",   // 例句（来自对话原句）
  "example_translation": "string", // 例句中文翻译
  "audio_url": "string",          // 发音音频URL
  "round_number": "number"        // 出现在对话的第几回合
}
```

### 4.2 字段说明
- **type枚举值**：
  - `word`: 单个单词，如 "order"、"salad"
  - `phrase`: 短语/词组，如 "ready to order"、"main course"、"I'd like to"

## 5. 测试题表 (Test)

### 5.1 主表结构
```json
{
  "test_id": "string",            // 测试题唯一标识
  "scene_id": "string",           // 关联场景ID
  "test_type": "string",          // 题型：choice/qa/open_dialogue
  "order": "number",              // 题目顺序
  "content": "object"             // 题目内容（根据题型不同结构不同）
}
```

### 5.2 题型1：多选一（选择合理回答）
```json
{
  "test_type": "choice",
  "content": {
    "question": "string",         // 场景核心问题
    "question_audio_url": "string", // 问题音频
    "options": [
      {
        "option_id": "string",    // 选项ID
        "text": "string",         // 选项文本
        "audio_url": "string",    // 选项音频
        "is_correct": "boolean",  // 是否正确
        "explanation": "string"   // 解析说明
      }
    ]
  }
}
```

### 5.3 题型2：问答题（语音回答）
```json
{
  "test_type": "qa",
  "content": {
    "question": "string",         // 场景核心问题
    "question_audio_url": "string",
    "standard_answer": "string",  // 标准回答文本
    "standard_answer_audio_url": "string",
    "answer_key_points": ["string"], // 回答关键点（用于AI判定）
    "acceptable_variations": ["string"] // 可接受的表达变体
  }
}
```

### 5.4 题型3：开放式对话
```json
{
  "test_type": "open_dialogue",
  "content": {
    "ai_role": "string",          // AI扮演的角色，如 "waiter"
    "ai_role_name": "string",     // 角色显示名称，如 "服务员"
    "scenario_description": "string", // 场景描述
    "dialogue_starters": ["string"], // AI可能的开场白（随机选择）
    "key_topics": ["string"],     // 对话应涵盖的关键话题
    "suggested_rounds": "number", // 建议对话回合数（非强制）
    "evaluation_criteria": {      // 评估标准
      "fluency": "number",        // 流畅度权重
      "accuracy": "number",       // 准确度权重
      "relevance": "number",      // 相关性权重
      "vocabulary": "number"      // 词汇使用权重
    }
  }
}
```

## 6. 用户学习记录表 (UserLearningRecord)

### 6.1 表结构
```json
{
  "record_id": "string",          // 记录唯一标识
  "user_id": "string",            // 用户ID
  "scene_id": "string",           // 场景ID
  "learning_status": "string",    // 学习状态：not_started/learning/completed
  "dialogue_completed": "boolean", // 对话学习是否完成
  "vocabulary_viewed": "boolean", // 是否查看过单词
  "test_results": ["TestResult"], // 测试结果列表
  "last_learning_time": "timestamp", // 最后学习时间
  "total_learning_duration": "number" // 累计学习时长（秒）
}
```

---

## 7. 测试结果表 (TestResult)

### 7.1 表结构
```json
{
  "result_id": "string",          // 结果唯一标识
  "user_id": "string",            // 用户ID
  "test_id": "string",            // 测试题ID
  "scene_id": "string",           // 场景ID
  "test_type": "string",          // 题型
  "user_answer": "object",        // 用户答案（结构根据题型不同）
  "is_correct": "boolean",        // 是否正确（选择题/判断题）
  "score": "number",              // 得分（问答题/开放对话，0-100）
  "ai_feedback": "string",        // AI反馈
  "completed_at": "timestamp"     // 完成时间
}
```

### 7.2 开放式对话结果详情
```json
{
  "result_id": "string",
  "test_type": "open_dialogue",
  "dialogue_transcript": [        // 对话记录
    {
      "speaker": "string",        // ai/user
      "text": "string",           // 文本内容
      "timestamp": "number"       // 时间戳（秒）
    }
  ],
  "quality_analysis": {           // 对话质量分析
    "overall_score": "number",    // 总分（0-100）
    "fluency_score": "number",    // 流畅度得分
    "accuracy_score": "number",   // 准确度得分
    "relevance_score": "number",  // 相关性得分
    "vocabulary_score": "number", // 词汇使用得分
    "strengths": ["string"],      // 优点列表
    "weaknesses": ["string"],     // 不足列表
    "suggestions": ["string"]     // 改进建议
  }
}
```

## 9. 数据库设计建议

### 9.1 关系型数据库（如MySQL/PostgreSQL）
- **Scene表**：存储场景基本信息
- **Dialogue表**：存储对话内容
- **DialogueRound表**：存储对话回合
- **QAAnalysis表**：存储问答解析
- **Answer表**：存储回答内容
- **Vocabulary表**：存储高频单词/短语
- **Test表**：存储测试题
- **UserLearningRecord表**：存储用户学习记录
- **TestResult表**：存储测试结果

### 9.2 NoSQL数据库（如MongoDB）
- 可以直接使用本文档中的JSON结构
- 每个场景作为一个Document存储
- 用户学习记录和测试结果单独存储

### 9.3 混合方案
- 核心数据（Scene、Dialogue、Test等）使用关系型数据库
- 用户行为数据（学习记录、测试结果）使用NoSQL
- 音频文件使用CDN存储，数据库只存URL

## 11. 数据生成建议

### 11.1 内容生成优先级
1. **日常场景**（5-8个）：问候、点餐、购物、问路、酒店入住、看病、交通出行、银行办理
2. **职场场景**（3-5个）：自我介绍、面试问答、会议讨论、邮件沟通、电话会议
3. **留学/考试场景**（3-5个）：雅思口语Part1、留学面签、校园生活、学术讨论、租房

### 11.2 内容质量标准
- 对话控制在3-5个回合，每个回合包含一问一答或多句对话
- 每个场景提供2-3个核心问答回合（带解析）
- 每个核心问答提供2-3种不同回答方式
- 高频单词/短语控制在8-15个
- 测试题包含：1道选择题、1道问答题、1个开放对话

### 11.3 音频资源
- 使用专业配音或TTS生成
- 音频格式：MP3，比特率128kbps
- 语速：正常语速（150-180词/分钟）
- 口音：美式或英式英语（保持一致）
